Relational Database Management System

    The goal of this project was to design and implement a small version of a relational database management system as well as implement a domain specific language similar to SQL to interact with the database. The user has capabilities to create, load, view, and drop tables as well as insert data into tables. The user can also filter data using select statements similar to SQL as well as perform natural inner and cartesian joins on the tables. This project was based on UC Berkeley's CS61B Spring 2017 database project.

Create Table

Here we can create a new table named football and populate it with the "insert into" command. We can also write the contents of the table to a .tbl file using the "store" command. Other previously created tables can be accessed (such as the records table) by using the load command.

Select Data

We can filter for the data we are looking for in the tables using a "select" statement and specifying the columns we would like returned. Columns can also be combined to form new columns using column expressions such as "Wins + Losses as Games from records" above. Conditional "where" clauses are supported to help the user better filter data.

Joins

If two tables are joined and they both have one or more columns matching columns with the other, a natural inner join is performed. If the two tables have no columns in common, a cartesian join will be performed.
