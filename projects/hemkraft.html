Hemkraft

    The goal of this project was to design and implement a database for use by a fictional organization named Hemkraft that wished to track characteristics of households and appliances within those households. After designing the relational database, a web application was developed to interact with the database. This project is from Georgia Tech's Database Systems Concepts & Design course and was built as a 5 student team.

EER Diagram

After being given a document of project specifications, the team developed the above Extended Entity Relationship Diagram modeling what data we were planning to store and the data relations.

Web App Homepage

After designing the database and creating the tables in Postgres, a web application was developed to interact with the database. Above is the homepage to the web application where you can select an option to enter data to be stored in the database or view reports from data stored in the database.

Reports

The web application supports the generation of reports that are built from data stored in the database. The web application is capable of generating the reports above.

Bathroom Statistics Report

Here is an example of a report, specifically the bathroom statistics report. The report returns information about the minimum, maximum and average amount of certain bathroom fixtures across all households in the database.

Model Search Report

For this report the user can search for specific manufacturers or models and see if they exist in the database. Entering text to search on in the text box at the top, the report will return all manufacturers or models that fully or partially match the users entered text. All matched manufatcurers or models are highlighted green.
